When Louise DeliversAdding Metadata to New Graphics

Follow these steps for each file Louise sends. The site uses canonical filenames — your job is to match her file to the right name, drop it in, and re-run the metadata script.

The workflow, one file at a time

  1. Check the file's actual format. Right-click the file → Properties. Look at "Type." Is it a JPEG or a PNG? Make sure the extension matches.
    Common gotcha: Canva sometimes exports JPEGs labeled as .png. If the extension doesn't match the actual format, ExifTool will refuse to write metadata. Rename the file to use the correct extension before proceeding.
  2. Rename the file to its canonical name. See the filename reference table below. Example: Louise sends KL_WH2U_Cover_FINAL_v3.jpg — rename to what-happened-to-us-age-of-ego-book-cover.jpg.
  3. Check pixel dimensions. Right-click → Properties → Details → Image width/height. If the dimensions differ from the record in the xlsx (column N "Pixel Dimensions"), you'll need to update the HTML later — note the new dimensions.
  4. Drop the file into the images folder. Replace the existing placeholder with the same name.
  5. Run the embed script on that single file. In PowerShell: 
    cd "C:\Users\marie\web graphics metadata"
python embed_metadata.py images\what-happened-to-us-age-of-ego-book-cover.jpg
    The script reads its canonical metadata from the xlsx, calls ExifTool, writes IPTC + XMP into the file.
  6. If dimensions changed: open the HTML file(s) that reference this image (see xlsx column O "Used On Pages"), find the <img src="..."> tag, update width="..." and height="..." to match the new dimensions.
  7. Commit and push to GitHub. Netlify auto-deploys within 2 minutes.

Running embed on ALL files at once

If Louise sends multiple files together, or if you want to re-embed everything (safe, idempotent), run:

cd "C:\Users\marie\web graphics metadata"
python embed_metadata.py images\

Script processes every file, skips anything not listed in the xlsx, reports success/failure per file.

When You're ReadyFull Launch Checklist

This is the sequence for flipping from "private working draft" to "visible to the world." Do this when Louise's graphics are substantially complete and you're ready to share the site publicly.

Before launch day

  • All (or enough) Louise graphics embedded with metadata
  • Click-test every internal link on every page — catch 404s before readers do
  • Mobile render audit — open every page on an actual phone, not just browser DevTools
  • Any remaining vertical pages written and added
  • Any prose edits you've been circling on (especially the "she keeps the master plan" drift in index.html prose about the Wise One)

Launch day

  1. Remove the noindex tags from every page. In GitHub, open the web file editor, OR clone locally and find-replace: 
    # Find this line in every .html file EXCEPT mapminder.html:
<meta name="robots" content="noindex, nofollow" />

# Delete it.
    Commit with message: "Launch: remove noindex tags for public launch"
  2. Create a sitemap. Either use a Netlify plugin (recommended: @netlify/plugin-sitemap) or generate a static sitemap.xml listing all public pages. Lives at kindredlights.org/sitemap.xml.
  3. Create robots.txt. A two-line file at repo root: 
    User-agent: *
Sitemap: https://kindredlights.org/sitemap.xml
  4. Set up Google Search Console. Visit search.google.com/search-console. Add kindredlights.org as a property. Verify ownership (Netlify makes this easy via DNS or a meta tag). Submit your sitemap.
  5. Test Open Graph previews. Before sharing, paste kindredlights.org into:
    • iMessage (to yourself)
    • Facebook post composer (without posting)
    • LinkedIn post composer (without posting)
    Each should show a proper title, description, and image. If any are broken, the og: meta tags on that page need adjustment.
  6. Do one final click-through. Every page. Every button. Mobile and desktop.
  7. Announce at your pace. Social media, email list, wherever feels right. No external pressure here.

Post-launch (nice-to-have, not urgent)

  • Analytics (Netlify Analytics $9/mo, or Plausible, or Google Analytics)
  • Custom 404 page (404.html in repo root — Netlify serves it automatically)
  • Convert book covers to .webp for faster load (~30% size reduction at same quality)
  • Accessibility audit (WCAG AA color contrast, full screen-reader pass)
  • Map Minder full parity upgrade (tooltips, JSON-LD, canonical metadata to match rest of site)

Working folder on your PC

  ├── images\              ← 25 canonical image files
  ├── embed_metadata.py    ← the embed script
  └── KL_GraphicsInventory_2026-04-21c.xlsx

ExifTool location

C:\Tools\exiftool\
  ├── exiftool.exe
  └── exiftool_files\

PATH is configured so exiftool works from any PowerShell window.

Verify ExifTool is working

exiftool -ver
# should print: 13.57 (or similar)

Check a file's current metadata

exiftool -Title -Description -Creator -Copyright -Keywords images\filename.jpg

Embed metadata on one file

python embed_metadata.py images\filename.jpg

Embed on all files

python embed_metadata.py images\

Dry run (see what would happen, change nothing)

python embed_metadata.py images\ --dry-run

GitHub workflow

The repository is kindredlights/kindredlights-website. Netlify watches the main branch and auto-deploys on every push.

  • Quick edits: use GitHub's web UI
  • Batch changes: clone locally, edit, push
  • Never push directly to main if there's anything uncertain — use a branch and pull request for safety
